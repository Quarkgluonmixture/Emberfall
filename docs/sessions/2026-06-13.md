## The Living Saga — chronicle legibility depth pass

**Problem (measured, not assumed).** A century probe bucketing chronicle
events by decade showed the world's *recorded history* is drowned: late-game
(y50+) macro events vs famine/plague/churn ran **332 : 23,299** (seed 1337),
**350 : 15,088** (7), **400 : 31,007** (42) — famine firing ~1,500/decade,
plague ~1,100. A famine every winter in every settlement isn't a catastrophe,
it's the Malthusian regulator: population grows to the food edge, winter
(food mult 0.45) tips it over, famine fires, spring heals it, repeat forever.

**Dead end that taught the real lesson.** First attempt suppressed crises at
the sim level (famine cooldown + higher hunger threshold, longer plague
immunity). Spam dropped 5–7× — **but civFell and rebirth went to ~0 across
all seeds.** The world got *more* static, just quieter. Finding: **famine/
plague are the engine of macro turnover, not just noise** — they supply the
morale/population pressure that collapses settlements and frees ruins for
rebirth. Suppressing them trades "illegible" for "dead." Reverted entirely.

**The actual fix — decouple the two roles, change presentation only.**
1. **FX/chronicle decoupling** (`fxLayer.ts`): the FX layer gated map juice on
   `importance >= 2`, so chronicle importance was overloaded — re-tiering the
   saga would have silently killed village sparkles / migration dust / plague
   flares. FX now gates on `KIND_FX` membership instead, independent of
   importance. Map stays lively no matter how the saga is tiered.
2. **Scale-based importance taxonomy** (`events.ts`, `simulation.ts`): retier
   so importance means *scale*, not *settlement event happened*:
   - **imp3 epochal spine** (starred): founding, warDeclared, treatySigned,
     capture, goldenAge, civFell, rebirth (~4/yr — clean).
   - **imp2 regional beats**: town rising, settlement collapse (was imp3!),
     succession, schism, wildfire, rivalry/peace/alliance/tradeOpened.
   - **imp1 texture**: village upgrade, migration/resettle, skirmish, famine,
     plague (primary + spread), flood, *End events, incidents.
   `collapse` 3→2 was the key spine fix — a hamlet abandoned no longer ranks
   equal to a civilization dying.
3. **Live feed** (`chroniclePanel.ts`): shows the last 9 *notable* (imp≥2)
   entries with an ember ★ on epochal ones, instead of raw churn. The history
   panel already filtered imp≥2 + starred imp3, so it benefits automatically.

**Result (final-chronicle imp≥2 = exactly what the panels surface; compaction
keeps every imp≥2):** surfaced saga **104→28/yr** (1337), **83→21** (7),
**154→44** (42) — 3.5–3.9× — with famine/plague now 0% of the surfaced set and
a clean ~4/yr starred epochal spine. Crisis *dynamics unchanged*: the changes
are RNG-neutral (importance is a literal arg to `pushEvent`, never drawn), so
the sim is **bit-identical to the prior baseline** — same lively macro
(civFell 5/0/5, reborn 3/0/3 over 150y on seeds 1337/7/42).

**Verify loop:** typecheck ✓ · 64/64 ✓ · 100-yr bit-identical stress on seeds

### Gotchas
- Probe methodology: counting chronicle events by per-decade `slice(before)`
  is **wrong under compaction** — `pushEntry` reindexes the array when it
  culls imp1 past the soft cap, so the offset goes stale and undercounts.
  Measure the *final chronicle* (compaction keeps every imp≥2, so it's an
  exact surfaced-history count) or instrument generation directly.
- Settlements rush to town tier and stay, so "town-only" gating ≈ no gating —
  a tier-based notability filter for famine/plague did almost nothing.
- `importance` was doing double duty (saga rank + FX trigger). Watch for that
  overload before re-tiering anything event-driven.
